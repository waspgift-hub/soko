SEHEMU YA 1: NOTIFICATIONS KUTOKA SERVER (index.js)
═══════════════════════════════════════════════════════════════

1. OTP AUTH (SMS tu)
   ─────────────────────────────────────────────
   Trigger    : Mtumiaji anaomba OTP kwenye /api/auth/send-otp
   Chaneli    : SMS (Meseji)
   Mpokeaji   : Mtumiaji anayeomba OTP
   SMS        : "Soko Vibe: OTP yako ni {otp}. Inaisha kwa dakika 10."


2. SEND-NOTIFICATION (API generic)
   ─────────────────────────────────────────────
   Trigger    : Simu kutoka Flutter kwenye /api/send-notification
                (like, order, comment, review, group chat, n.k.)
   Chaneli    : In-app (Firestore notifications) + FCM
   Mpokeaji   : userId yoyote (kutoka kwa mwita)
   Title/Body : Inategemea mwita
   Aina       : Inategemea data.type


3. SEND-BULK-NOTIFICATION (Admin)
   ─────────────────────────────────────────────
   Trigger    : Admin anabonyeza "Send to All" kwenye dashboard
   Chaneli    : FCM multicast (hadKYC tokens 500/group)
   Mpokeaji   : Wateja wote wenye FCM token
   Title/Body : Kuanzia admin input


4. MONGIKE WEBHOOK — BOOST PAMENT SUCCESS
   ─────────────────────────────────────────────
   Trigger    : Mongike anaripoti malipo ya boost yamekamilika
   Chaneli    : In-app + FCM + SMS + Broadcast kwa WOTE
   Mpokeaji   : Muuzaji aliyenunua boost
   Title      : "✅ Boost imewashwa!"
   Body       : "Bidhaa yako imepandishwa kwa daraja la {tier} kwa siku {days}."
   SMS        : "Soko Vibe: Malipo ya Boost ya TZS {amount} yamefanikiwa!"
   Aina       : boost → FCM Channel: general_notifications_v4
   Pia        : notifyBoostBroadcast() inatahadharisha WATEJA WOTE


5. MONGIKE WEBHOOK — PURCHASE SUCCESS
   ─────────────────────────────────────────────
   Trigger    : Mongike anaripoti malipo ya bidhaa yamekamilika
   Chaneli    : In-app + FCM + SMS
   Mpokeaji   : Muuzaji (seller)
   Title      : "Umepata Mauzo!"
   Body       : "{productName} imeuzwa. TZS {amount} zimewekwa escrow."
   SMS        : "Soko Vibe: Oda #{orderId} imelipiwa! Fedha ipo salama Escrow."
   ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─
   Mpokeaji   : Mnunuzi (buyer)
   Title      : "Malipo Yamekamilika!"
   Body       : "Malipo ya {productName} yamepokelewa. Thibitisha upokeaji."
   SMS        : "Soko Vibe: Malipo ya TZS {price} kwa Oda #{orderId} yamepokelewa."
   Aina       : order / escrow_confirm → FCM Channel: payments_notifications_v4


6. MONGIKE WEBHOOK — PAYMENT FAILED
   ─────────────────────────────────────────────
   Trigger    : Mongike anaripoti malipo yameshindikana
   Chaneli    : In-app + FCM + SMS
   Mpokeaji   : Mnunuzi (buyer)
   Title      : "Malipo Yameshindikana"
   Body       : "Malipo ya {productName} hayakukamilika. Jaribu tena."
   SMS        : "Soko Vibe: Malipo ya {productName} hayakukamilika."
   Aina       : payment_failed → FCM Channel: payments_notifications_v4


7. DISPATCH — Muuzaji anasafirisha bidhaa
   ─────────────────────────────────────────────
   Trigger    : Muuzaji anabonyeza "Dispatch" kwenye /api/escrow/dispatch
   Chaneli    : In-app + FCM + SMS
   Mpokeaji   : Mnunuzi (buyer)
   Title      : "📦 Bidhaa Imesafirishwa!"
   Body       : "{productName} imesafirishwa. Angalia proof of delivery."
   SMS        : "Soko Vibe: Mzigo wa Oda #{orderId} umesafirishwa kupitia basi la {busName} ({plateNumber})."
   Aina       : dispatched → FCM Channel: payments_notifications_v4


8. ESCROW RELEASE — Mnunuzi anathibitisha upokeaji
   ─────────────────────────────────────────────
   Trigger    : Mnunuzi anabonyeza "Confirm Delivery" /api/escrow/release
   Chaneli    : In-app + FCM + SMS
   Mpokeaji   : Muuzaji (seller)
   Title      : "Escrow Imefunguliwa!"
   Body       : "Mnunuzi amethibitisha upokeaji wa {productName}. TZS {amount} zimewekwa salio."
   SMS        : "Soko Vibe: Mteja amethibitisha kupokea mzigo #{orderId}. TZS {amount} zimetolewa Escrow."
   ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─
   Mpokeaji   : Mnunuzi (buyer)
   Title      : "Umethibitisha Upokeaji"
   Body       : "Umethibitisha kuwa umepokea {productName}. Pesa zimefunguliwa kwa muuzaji."
   Aina       : escrow_release / delivery_confirmed → FCM: payments_notifications_v4


9. CANCEL — Mnunuzi anaghairi order
   ─────────────────────────────────────────────
   Trigger    : Mnunuzi anaghairi order kabla ya dispatch
   Chaneli    : In-app + FCM
   Mpokeaji   : Mnunuzi (buyer)
   Title      : "💰 Pesa Zimerudishwa Kamili"
   Body       : "Refund kamili ya TZS {amount} kwa {productName} imetumwa."
   Mpokeaji   : Muuzaji (seller)
   Title      : "❌ Oda Imeghairiwa"
   Body       : "{productName} imeghairiwa na mnunuzi."
   Aina       : refund / cancelled → FCM: payments_notifications_v4


10. DISPUTE — Mnunuzi anafungua mgogoro
    ─────────────────────────────────────────────
    Trigger    : Mnunuzi anafungua mgogoro
    Chaneli    : In-app + FCM + Admin notification
    Mpokeaji   : Muuzaji (seller)
    Title      : "⚖️ Mgogoro Umefunguliwa"
    Body       : "Mnunuzi amefungua mgogoro kwa {productName}."
    Mpokeaji   : Mnunuzi (buyer)
    Title      : "⚖️ Mgogoro Umefunguliwa"
    Body       : "Tumepokea mgogoro wako. Admin atakagua."
    Mpokeaji   : Admins WOTE (kupitia notifyAdmins)
    Title      : "⚖️ Mgogoro Mpya Unahitaji Uamuzi"
    Aina       : disputed → FCM: payments_notifications_v4


11. ADMIN RESOLVE DISPUTE
    ─────────────────────────────────────────────
    Trigger    : Admin anatoa uamuzi (release au refund)
    Chaneli    : In-app + FCM
    a) Uamuzi = release
       Seller   : "⚖️ Uamuzi wa Mgogoro" — "Admin ameamua pesa zikutolee."
       Buyer    : "⚖️ Uamuzi wa Mgogoro" — "Admin ameamua pesa zitolewe kwa muuzaji."
    b) Uamuzi = refund
       Buyer    : "💰 Pesa Zimerudishwa Kamili"
       Seller   : "❌ Mgogoro Umekamilika" — "{productName} imerefundiwa mnunuzi."
    Aina       : dispute_resolved / refund → FCM: payments_notifications_v4

23. TRANSACTION LISTENER: pending → escrow_hold
    ─────────────────────────────────────────────
    Trigger    : Firestore transaction.status = escrow_hold
    Chaneli    : FCM + SMS
    FCM Buyer  : "Payment Received – Escrow Held"
                 "Your payment for {productName} is held in escrow."
    SMS Buyer  : "Soko Vibe: Malipo ya TZS {total} kwa Oda #{orderId}
                  yamepokelewa na kuwekwa salama Escrow."
    SMS Seller : "Soko Vibe: Oda #{orderId} imelipiwa! Fedha ipo salama
                  Escrow. Tafadhali kamilisha usafirishaji stendi."
    Aina       : payment → FCM: payments_notifications_v4


24. TRANSACTION LISTENER: escrow_hold → dispatched
    ─────────────────────────────────────────────
    Trigger    : Firestore transaction.status = dispatched
    Chaneli    : SMS tu (FCM inafanywa na /api/escrow/dispatch endpoint)
    SMS Buyer  : "Soko Vibe: Mzigo wa Oda #{orderId} umesafirishwa
                  kupitia basi la {busName} ({plateNumber})."


25. TRANSACTION LISTENER: dispatched → delivered
    ─────────────────────────────────────────────
    Trigger    : Firestore transaction.status = delivered
    Chaneli    : SMS tu (FCM inafanywa na /api/escrow/release endpoint)
    SMS Seller : "Soko Vibe: Mteja amethibitisha kupokea mzigo #{orderId}.
                  TZS {amount} zimetolewa Escrow na kuwekwa kwenye pochi yako."


26. TRANSACTION LISTENER: any → failed
    ─────────────────────────────────────────────
    Trigger    : Firestore transaction.status = failed
    Chaneli    : FCM + SMS
    FCM Buyer  : "Malipo Yameshindikana"
                 "Malipo ya {productName} hayakukamilika. Fungua app ili ujaribu tena."
    SMS Buyer  : "Soko Vibe: Malipo ya {productName} hayakukamilika."
    Aina       : payment_failed → FCM: payments_notifications_v4


27. TRANSACTION LISTENER: any → refunded
    ─────────────────────────────────────────────
    Trigger    : Firestore transaction.status = refunded
    Chaneli    : FCM + SMS
    FCM Buyer  : "Fedha Zimerudishwa"
                 "Fedha za {productName} zimerudishwa kwenye akaunti yako."
    SMS Buyer  : "Soko Vibe: Fedha za {productName} (Oda #{orderId})
                  zimerudishwa kwenye akaunti yako."
    Aina       : refund → FCM: payments_notifications_v4


28. CHAT MESSAGE LISTENER

29. PRODUCT LISTENER — Bidhaa mpya
